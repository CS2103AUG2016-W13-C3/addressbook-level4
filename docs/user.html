About

CommanDo is a task scheduler that helps you to manage the flood of things you need to do (to-do items). This application is a primarily text-based – almost all input will be through commands, using the text box at the bottom. In this guide, we will be splitting the to-do items into 2 groups: events and tasks, which are identified by the number of date time fields they require.

Quick Start

  1. Ensure you have Java version 1.8.0_60 or later installed in your Computer (earlier versions of Java 8 will not work).

  2. Download the latest commando.jar from the releases tab.

  3. Copy the file to the folder you want to use as the home folder for CommanDo.

  4. Double-click the file to start the application. The GUI should appear in a few seconds.

  5. Type help and pressing Enter will open up a link to this User Guide. This is the place where you enter commands.

  6. Refer to the Commands section below for details of each command.

    Commands

The first keyword in your line of command will tell the application what to do, with optional parameters after it. The following legend describes the syntax of the code used in the remainder of this guide:

  1. <> denotes fields to fill in
  2. [] denotes optional
  3. ... denotes 0 or more

Viewing Help: help

If you want to check the format of a command, you may refer to the command cheatsheet in user guide or use help command.

Format: 
>> help [<command>]

The command help shows a complete list of commands and examples of how to use them properly. If [<command>] is also given, examples of how to use that command will be shown.

Adding To-Do Items: add

As a new user, the first thing you would do is to add a to-do item. To add a to-do item in the list, you have to follow the format of add command.

There are some rules on the format of to-do items in In CommanDo To-do items must contain at minimum, a description of what it is. They may contain additional information, like tags, or a time frame. In CommanDo, to-do items are categorised into two types: Events and Tasks. An Event contains a start datetime and an end datetime which represent the time slot and duration for that event while a Task contains a end datetime which represents the deadline of the task.

Events:

Events are activities that are happening within a time frame. In CommanDo, they are to-do items with a from keyword and a to keyword. In particular, the from keyword must come before the to keyword.

Format: 
>> add <description> from <start_datetime> to <end_datetime> [#<tag>...]

There are many ways in which dates and times can be written. Some example would be 5 Jan 2016 and 28/12/2016 (DD/MM/YYYY). The year can dropped for convenience. The full list of accepted formats in CommanDo are described in FAQ section below.

Sample input:
>> add meeting with boss from 5 Sep 12:30 to 17:30 #important
>> add meeting with Dr Tan from 30 Sep 9:00 to 10:00 #work #important
>> add wild party in hostel room from 31 Dec 2016 to 1 Jan 2017 #yolo

Tasks:

Tasks are to-do items that may be done in your own time, or have a deadline. In CommanDo, they are to-do items that may have a due date. A task can be something you need to do by a certain date or time, or just something you can do at your own time.

Format: 
>> add <description> [by <due_datetime>] [#<tag>...]
Sample input:
>> add do Homework 10 by 23 Sep 
>> add release V0.5 by 7 Nov 2359 
>> add finish FYP by tomorrow 2359 
>> add shop for groceries: banana, pineapple, watermelon #housework 
>> add watch Lord of the Rings

Editing To-Do Items: edit

If you want to edit the details of a to-do item, the command edit edits existing to-do items by accessing their index, and providing the fields to replace. At least one of the optional parameters should be included, if not the command does nothing. If there are tags in your input, they will replace the tags stored for the target to-do item.

Events:

To edit the details of an event, at least one of the optional parameters shown below must be provided to make a meaningful edit.

Format: 
>> edit <index> [description] [from <start_datetime>] [to <end_datetime>] [#<tag>...]
Sample input:
>> add meeting with boss from today 1300 to today 1500 
>> edit 1 from today 1400 to today 1600 
>> edit 2 #projectABC

Tasks:

To edit the details of a task, at least one of the optional parameters below must be provided to make a meaningful edit. You can change or add a due date-time to a task. You can also remove the due date-time from a task by entering nothing after the by keyword.

Format: 
>> edit <index> [description] [by <due_datetime>] [#<tag>...]
Sample input:
>> add plan shopping trip #lunch
>> edit 2 shopping trip to IKEA
>> edit 2 by 6 feb
>> edit 2 by
>> add watch Inception
>> edit 3 watch Gravity
>> edit 3 #family

Deleting To-Do Items: delete

If you have completed and wanted to delete a to-do item, you may use delete command. This command deletes an exsiting event or task by accessing the index of targeted to-do item and delete it from the list accordingly.

Format: 
>> delete <index>

Clearing all To-Do Items: clear

If you want to reset the CommanDo, you may use clear command which clears all to-do items, essentially wiping your save file. Use with caution.

Format: 
>> clear

Finding To-Do Items: find

You have already built a long list of to-do items and want to read the details of one of them. You can use find command to achieve this. This command searches and lists all the to-do items which have descriptions or tags containing all of the given keywords.

Format: 
>> find <keyword>...

Here are some instructions on how to use the find command:

  • If you do not key in any keywords after find, it will show you all to-do items
  • Case insensitive. – Chicken Egg will match the keyword chicken egg
  • Ordering of the keywords does not matter. – Chicken Egg will match the keywords Egg Chicken
  • Parts of words will be matched. – chickens will match the keyword chicken, whereas chicken will not match the keyword chickens
  • All keywords must be codesent. – find chicken egg will match the keywords chicken lay egg but not keywords bird lay egg

Setting Save Location: set

You want to save the data file somewhere else you want. By typing set command, it changes the location where the to-do list is saved on the local file system to the given location / path.
A file path can be absolute (C:/home/sally/statusReport) or relative to the commando.jarfile (joe/foo/bar). This applies to the next two file location relevant commands, i.e. export and import.

Format: 
>> set storage <path>

Exporting: export

If you want to copy the data file to a local destination, you don't need to go into the folder containing your data file. Instead, export command exports the current to-do list as a save data file at the given location / path which saves your time.

Format: 
>> export <path>

Importing: import

If you already have existing data file and want you use it in the CommanDo, the import command imports a valid save data file at the given location / path and overrides the current to-do list with the imported to-do list.

Format: 
>> import <path>

Undoing: undo

You accidently deleted a wrong to-do item and you want to cancel your last modification. The CommanDo does not support ctrl + z but the undo command undoes the last add, edit, delete, clear, import or set storagecommand. You may use it to recover your data.

Format: 
>> undo

Exiting the Application : exit

Finally you have done all your work and want to close the program. You can either press the cross button or directly use exit command provided in CommanDo. This command saves all changes to to-do list and closes the application. You cannot undo your last command after closing and restarting the program.

Format: 
>> exit

FAQ

Saving data

Your to-do list is automatically saved to your file system after any command that changes the data. There is no need to save manually and there is no save command supported by CommanDo.

Sharing data across devices

Use the export command to generate a save data file, transfer the file to a 2nd device, and use the import command on the 2nd device to import your to-do list from that save file. Alternatively, you can use a cloud-syncing service (e.g. Dropbox) and the set storage command to sync the to-do list automatically by setting to the same storage file on both devices.

Supported date-time formats

CommanDo supports a wide variety of date and time formats.

A valid date-time is defined as:

  1. A valid date.
  2. A valid date followed by a valid time.

where examples of valid dates include:

Standard Date Notation With words Presets
10/02/2016 (DD/MM/YYYY) 10 Feburary 2016 today
10/02/16 (DD/MM/YY) 10 Feb 2016 tomorrow
10/02 (DD/MM) 10 Feb next Wednesday
10th Feb n days after (where n is bigger than 0)
Feb 10

and examples of valid time include:

Standard Time Notation Omitted Colons Presets
19:00 (hh:mm) 2359 morning (0900h by default)
9am (hh)pm/am 0911h afternoon (1200h by default)
11:59pm (hh:mm)pm/am evening (1900h by default)
