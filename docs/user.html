What is CommanDo?

You have a ton of things to do, but you really hate the amount of mouse-clicking and the complicated user interfaces you have to go through to edit a to-do list in most to-do list management software...

Meet CommanDo - it is the to-do list manager that keeps things clean and allows you to edit your to-do list swiftly by typing out commands, mouse-free. Commands in CommanDo are simple and intuitive, while providing a bit of flexibility in syntax and customizability. CommanDo recognizes and actively categories to-dos into 2 groups: events (to-dos with a description and a time window) and tasks (to-dos with just a description, accompanied by an optional deadline).

This guide explains how to install, use and troubleshoot CommanDo.

Getting Started

  1. Ensure that you have Java version 1.8.0_60 or later on your device (here's how). Instructions on how to install latest version of Java can be found here.
  2. Download the latest commando.jar from CommanDo's releases tab on Github.
  3. Copy commando.jar to the folder you want to use as the home folder for CommanDo.
  4. Double-click the file to start the application. CommanDo's window should appear in a few seconds:
  5. Type help and press Enter - a window will open up to show this exact guide. This is how you'll be entering commands in CommanDo. Refer to the Commands section below for a whole list of commands.

Commands

Viewing Help: help

Command Format:

>> help [<command_word>]
Understanding Command Formats For each command, the first word is the command word that specifies the type of the command. What follows after are fields to fill in for that command. The following legend describes the syntax we used to specify command formats:
  1. >> denotes a field to fill in
  2. [] denotes that the fields within it are optional
  3. ... denotes that the preceding field can be repeated (multiple values)

Examples:

>> help
What it does: Opens up this user guide. 
>> help add
What it does: Opens up this user guide, jumping to the section on the add command. 
>> help cheatsheet
What it does: Opens up this user guide, jumping to the Cheatsheet section.

Adding To-Do Items: add

The very first thing you'll need to do is to add a to-do. Each to-do item contains minimally a description, and can be have multiple tags and/or time constraint / interval associated with it. To-do items are categoried into 2 types: Events (shown in the left panel) and Tasks (shown in the right panel):

Adding Events

Command Format: 
>> add <description> from <start_datetime> to <end_datetime> [#<tag>...]

An Event is an activity happening within a time frame, containing start_datetime and end_datetime fields to represent the time frame. You can also add tags prefixed with # to help organise your to-do list. On the interface of CommanDo, every Event from the current day onwards is listed on the left panel, in chronological order:

Datetime Formats CommanDo is pretty flexible in parsing your datetime fields, accepting formats like 5 Jan 2016 2300 and 28/12/2016 12pm (where date is in format DD/MM/YYYY). The year can be dropped for convenience, if you mean the current year. For the full list of usable datetime formats, refer to the section on supported datetime formats below.
Examples: 
>> add shopping with wife from 5 Sep 1230 to 1730 #important
>> add meeting with Dr Tan from 30 Sep 900 to 1000 #work #important
>> add wild party in hostel room from 31 Dec 2016 to 1 Jan 2017 #yolo

Adding Tasks

Command Format: 
>> add <description> [by <due_datetime>] [#<tag>...]

A Task, on the other hand, is an activity that can be done at your own time or by some deadline. Therefore, it has an optional due_datetime field. Similarly, tags can be added to a Task. On the interface of CommanDo, every Task is listed on the right panel, with those with deadlines prioritized at the top in chronological order:

Escaping command keywords To parse commands properly, CommanDo uses a set of keywords such as by, from, and to. CommanDo tries its best to determine whether a word is supposed to be a keyword or part of a field, but in case of ambiguity, surround the word with "" to tell CommanDo that it is to be interpreted literally. For example: 
>> add code parser to parse "by" tomorrow by today
Examples: 
>> add do Homework 10 by 23 Sep
>> add release V0.5 by 7 Nov 2359
>> add finish FYP by tomorrow 2359
>> add shop for groceries: banana, pineapple, watermelon #housework
>> add watch Lord of the Rings

Editing To-Do Items: edit

More often than not, you'll need to edit the fields of a to-do item after adding it, such as when you made a mistake, when an Event gets postponed, or when you receive more description for a Task.

Editing Events

Command Format: 
>> edit <index> [<description>] [from <start_datetime>] [to <end_datetime>] [#<tag>...]
Index of To-Do Item To refer to an existing to-do item in CommanDo, its Index is used, which is the number visually tagged to a to-do item on the interface. This clarified in the following figure:

By including suitable fields and their keywords, you'll be able to edit the description, time slot and tags of an Event with Index of <index>. Note that if tags are provided in an edit, the old tags will be completely replaced by the new ones.

Examples: 
>> add meeting with boss from today 1300 to today 1500
>> edit 1 from today 1400 to today 1600 #important
>> edit 1 #projectABC
What it does: adds an Event, then changes its time slot and adds a tag to it, then replaces a tag with another.

Editing Tasks

Command Format: 
>> edit <index> [<description>] [by <due_datetime>] [#<tag>...]

By including suitable fields and their keywords, you'll similarly be able to edit the description, deadline and tags of a Task. In addition, you can remove the deadline by entering nothing after the by keyword.

Examples: 
>> add plan shopping trip #lunch
>> edit 1 shopping trip to IKEA
>> edit 1 by 6 feb
>> edit 1 by
What it does: adds an Task with a tag, then changes its description, then adds a deadline to it, then removes that deadline. 
>> add watch Inception
>> edit 1 watch Gravity
>> edit 1 #family
What it does: adds an Task, then changes its description, then adds a tag to it.

Deleting To-Do Items: delete

If you want to delete a to-do item, whether it is an Event or a Task, use the delete command.

Command Format: 
>> delete <index>

Finishing To-Do Items: finish

After finishing a to-do item, mark it as done with the finish command! That to-do item will disappear from the interface of CommanDo. Note that this is an alias of the delete command.

Command Format: 
>> finish <index>

Finding To-Do Items: find

So you have already built a long list of to-do items, and want to search out the details of one of them by either its description or its tags. You can use the find command to achieve this.

Command Format: 
>> find <keyword>...

After doing the search, use the find command again without any keywords to show all to-do items.

Some details on how find command works:

  • Both Events and Tasks will be searched, in their separate panels.
  • Search is case insensitive – keywords Chicken Egg will match chicken egg.
  • Ordering of the search keywords does not matter – keywords Chicken Egg will match Egg Chicken.
  • Search keywords will match parts of words – keyword chicken will match chickens, whereas keyword chickens will not match chicken.
  • All search keywords must be matched – keywords chicken egg will match chicken lay egg but not bird lay egg.

Examples: 
>> find lunch meeting
What it does: Filters to show only Events and Tasks with description or tags matching both lunch and meeting. 
>> find
What it does: Clears previous find command and shows all to-do items.

Clearing Your To-Do List: clear

If you want to reset your to-do list, use the clear command. Use with caution.

Command Format: 
>> clear

Setting Save Location: store

Maybe you want to save your to-do list in your Dropbox or OneDrive folder that is synced with the cloud. The store command lets you do exactly that - it changes the location where the to-do list data is saved on your local file system.

Command Format: 
>> store <file_path>
File Paths A file path can be absolute (C:/Users/Jim/Dropbox/work/todolist.xml) or relative to the commando.jar file (../../Dropbox/work/todolist.xml). You can use any file extension you wish, but ending with .xml is recommended because the date file is in XML.
Examples: 
>> store ../../Dropbox/work/todolist.xml
What it does: Changes the default file path to store the to-do list data to ../../Dropbox/work/todolist.xml relative to your commando.jar, and saves the latest data to that file.

Exporting: export

If you want to copy your to-do list data file in your current save location to another local destination, there's no need to manually do so. Instead, using the export command saves a copy of the current to-do list data to the specified file at <file_path>.

Command Format: 
>> export <file_path>

Importing: import

If you already have an existing to-do list data file (possibly from your other devices) and want you use it in CommanDo, the import command can import a valid data file. Note that it overrides the current to-do list with the imported to-do list! Command Format: 

>> import <file_path>

Undoing: undo

Scenario: you deleted a to-do item by accident and you want to undo your last modification. Although CommanDo does not support Ctrl + Z, the undo command does the same: the most recent add, edit, delete, clear, import or storecommand would be reverted. Multiple undo commands can be used in succession, but note that the capacity to undo is lost if you close the application.

Command Format: 
>> undo

Redoing: redo

Scenario: you deleted a to-do item by accident, cleverly used undo to restore that to-do item, but eventually realised that to-do item isn't worth keeping after all. The redo command reverts the most recent undo, can be used in sucession, but similarly loses its capacity when the application is closed.

Command Format: 
>> redo

Exiting the Application: exit

Finally, you are done with managing your to-do list and want to close the program. You can either press the cross button or directly use exit command provided in CommanDo. This command closes the application, with all the changes to your to-do list saved.

Command Format: 
>> exit

FAQ & Troubleshooting

Clicking on commando.jar doesn't work or gives an error.

You'll need to install the latest version of Java. Refer to the installation guide.

How do I save my to-do list after modifying it?

Your to-do list is automatically saved to your file system after any command that changes the data. There is no need to save manually.

Is sharing across devices supported?

CommanDo doesn't directly support syncing of the to-do list across devices. Instead, use the export command to generate a save data file, transfer the file to another device, and use the import command on the 2nd device to import your to-do list from that save file. Alternatively, you can use a cloud-syncing service (e.g. Dropbox) and the store command to sync the to-do list automatically by setting the save location to the same file on both devices.

My commands keep giving me an error and I have no idea what is wrong.

When the format of your command is invalid, CommanDo will tell you the nature of the error and display the correct command format in the command feedback box just below the command input. CommanDo relies on a specific command syntax to understand your commands, in which the first word in your command is the command word. Based on the command word, refer to the appropriate section under Commands, read up on the command format for that command, and try out some of the given sample commands.

I keep forgetting how to use certain commands.

The help command (explained in the section above) can be used to quickly summon this guide or a relevant section of this guide when you need it. Alternatively, you can just try typing your command - if it is invalid, CommanDo will display the correct command format for that command in the command feedback box just below the command input. This might be helpful for you to correctively learn the command format.

What date & time formats can I use in my commands?

CommanDo supports a wide variety of date and time formats, backed by the awesome date parser library Natty.

A valid datetime is defined as:

  1. A valid date
  2. A valid date followed by a valid time
  3. A valid time (if it is not the first datetime appearing in the command)*

* In this case, the datetime will be interpreted to be on the same date as the first datetime

Valid Dates

With Slashes With Month Word Relative Dates
10/02/2016 (DD/MM/YYYY) 10 Feburary 2016 today
10/02/16 (DD/MM/YY) 10 Feb 2016 tomorrow
10/02 (DD/MM) 10 Feb next Wednesday
10th Feb n days after (where n is bigger than 0)
Feb 10

Valid Times

24 hour notation Using am/pm Presets
19:00 (hh:mm) 11:59pm (hh:mm)pm/am morning (0900h by default)
0911h 9am (hh)pm/am afternoon (1200h by default)
2359 evening (1900h by default)
